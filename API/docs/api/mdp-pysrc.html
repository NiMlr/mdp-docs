 1  # Modular toolkit for Data Processing (MDP) 
 2  """ 
 3  Modular toolkit for Data Processing (MDP) is a data processing 
 4  framework written in Python. 
 5   
 6  From the user's perspective, MDP consists of a collection of trainable 
 7  supervised and unsupervised algorithms or other data processing units 
 8  (nodes) that can be combined into data processing flows. Given a 
 9  sequence of input data, MDP takes care of successively training or 
10  executing all nodes in the flow. This structure allows to specify 
11  complex algorithms as a sequence of simpler data processing steps in a 
12  natural way. Training can be performed using small chunks of input 
13  data, so that the use of very large data sets becomes possible while 
14  reducing the memory requirements. Memory usage can also be minimized 
15  by defining the internals of the nodes to be single precision. 
16   
17  The base of readily available algorithms includes Principal Component 
18  Analysis, two flavors of Independent Component Analysis, Slow Feature 
19  Analysis, Gaussian Classifiers, Growing Neural Gas, Fisher 
20  Discriminant Analysis, and Factor Analysis. 
21   
22  From the developer's perspective, MDP is a framework to make the 
23  implementation of new algorithms easier. The basic class 'Node' takes 
24  care of tedious tasks like numerical type and dimensionality checking, 
25  leaving the developer free to concentrate on the implementation of the 
26  training and execution phases. The node then automatically integrates 
27  with the rest of the library and can be used in a flow together with 
28  other nodes. A node can have multiple training phases and even an 
29  undetermined number of phases. This allows for example the 
30  implementation of algorithms that need to collect some statistics on 
31  the whole input before proceeding with the actual training, or others 
32  that need to iterate over a training phase until a convergence 
33  criterion is satisfied. The ability to train each phase using chunks 
34  of input data is maintained if the chunks are generated with 
35  iterators. Moreover, crash recovery is optionally available: in case 
36  of failure, the current state of the flow is saved for later 
37  inspection. 
38   
39  MDP has been written in the context of theoretical research in 
40  neuroscience, but it has been designed to be helpful in any context 
41  where trainable data processing algorithms are used. Its simplicity on 
42  the user side together with the reusability of the implemented nodes 
43  make it also a valid educational tool. 
44   
45  As its user base is steadily increasing, MDP appears as a good 
46  candidate for becoming a common repository of user-supplied, freely 
47  available, Python implemented data processing algorithms. 
48   
49  http://mdp-toolkit.sourceforge.net 
50  """
